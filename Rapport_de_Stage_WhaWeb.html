1. Introduction Générale

À l'ère de la transformation numérique, la communication instantanée est devenue le pilier central de la relation client. WhatsApp, avec plus de 2 milliards d'utilisateurs, s'impose comme l'outil privilégié des entreprises pour interagir avec leur audience.

Le projet WhaWeb s'inscrit dans cette dynamique. Il vise à transformer un simple compte WhatsApp en un véritable outil de CRM (Customer Relationship Management) et de marketing automatisé. Durant ce stage chez FirstOne Media, j'ai eu la responsabilité de concevoir et développer une application Full Stack capable de gérer des contacts, de lancer des campagnes de messagerie et d'automatiser les réponses via l'Intelligence Artificielle.

3. Analyse des Besoins et Conception

3.1. Problématique

Les entreprises font face à trois défis majeurs sur WhatsApp :

  1. La difficulté de gérer manuellement des centaines de contacts.
  2. L'impossibilité d'envoyer des messages groupés sans risquer le bannissement ou la perte de temps.
  3. La nécessité d'être disponible 24h/24 pour répondre aux questions récurrentes des clients.

3.2. Cahier des Charges

Les fonctionnalités prioritaires identifiées étaient :

  • Authentification sécurisée : Connexion via QR Code WhatsApp.
  • Gestion de Base de Données : Stockage des contacts et de l'historique.
  • Importation Excel : Gain de temps massif pour les équipes marketing.
  • Dashboard Analytique : Suivi des statistiques d'envoi et de lecture.
  • Moteur IA : Réponses automatiques basées sur une base de connaissances.

4. Environnement Technique

Pour assurer scalabilité et performance, j'ai opté pour la Stack MERN :

  • MongoDB : Base de données NoSQL flexible pour gérer des documents JSON (contacts, logs).
  • Express.js : Framework backend pour Node.js permettant de créer des APIs RESTful rapides.
  • React.js : Bibliothèque frontend pour créer une interface utilisateur réactive et modulaire.
  • Node.js : Environnement d'exécution JavaScript côté serveur.

Outils complémentaires :

  • Tailwind CSS : Pour un design "utility-first" moderne.
  • OpenAI SDK : Pour l'intégration des modèles GPT.
  • whatsapp-web.js : Pour l'interaction avec le protocole WhatsApp.

5. Réalisation et Mise en Œuvre

5.1. Développement Backend

Le cœur du système repose sur Node.js. J'ai structuré le serveur en plusieurs modules : routes, modèles et contrôleurs. Un aspect crucial a été la gestion des événements WhatsApp en temps réel.

5.2. Architecture MongoDB

J'ai conçu trois schémas principaux avec Mongoose :

  • Contact : Nom, Téléphone, Tags.
  • MessageLog : ID message, Type (SENT/RECEIVED), Statut (READ/REPLIED), Timestamp.
  • AIConfig : Clé API, System Prompt, Knowledge Base.

5.3. Interface Utilisateur (React)

L'interface a été conçue pour être intuitive. J'ai implémenté un système d'onglets (Dashboard, Contacts, Campagnes, IA Config) permettant une navigation fluide. Les composants sont stylisés avec Tailwind pour garantir une esthétique professionnelle.

5.4. Innovation : Importation Excel Massive

L'une des fonctionnalités les plus avancées est le module d'importation. En utilisant la librairie xlsx, j'ai permis à l'utilisateur de charger des milliers de contacts en quelques secondes.

Le processus inclut :

  1. Lecture du fichier binaire côté client.
  2. Parsing des colonnes (Nom, Téléphone).
  3. Nettoyage des données (suppression des espaces, formatage international).
  4. Envoi groupé au serveur (Bulk Insert) pour optimiser les performances de la base de données.

5.5. L'Intelligence Artificielle au service du Marketing

L'assistant IA ne se contente pas de répondre. Il analyse l'intention du client. Si un client demande un prix, l'IA consulte la "Knowledge Base" configurée sur le dashboard et répond avec précision en utilisant le modèle GPT-4o.

6. Défis Techniques et Solutions

Durant le développement, j'ai été confronté à des défis complexes :

A. Le problème de l'identifiant WhatsApp (LID)

Défi : WhatsApp utilise parfois des IDs internes (@lid) qui diffèrent des numéros de téléphone réels (@c.us), ce qui rendait le calcul du taux de réponse impossible.

Solution : J'ai développé un Universal Matcher. Cet algorithme extrait les 8 derniers chiffres de chaque identifiant et effectue une recherche par expression régulière (Regex) dans la base de données pour lier le message entrant au message envoyé, garantissant des statistiques 100% exactes.

B. Persistance de la session

Défi : Le QR Code devait être scanné trop souvent.

Solution : Implémentation de LocalAuth pour sauvegarder les données de session Puppeteer dans un dossier local sécurisé, permettant une reconnexion automatique du serveur.

8. Conclusion et Perspectives

Ce stage chez FirstOne Media a été une expérience transformatrice. J'ai pu mener un projet de A à Z, de la compréhension du besoin client jusqu'à l'implémentation de fonctionnalités d'Intelligence Artificielle de pointe.

Le projet WhaWeb remplit tous les objectifs fixés. Pour la suite, nous envisageons d'ajouter le support des messages multimédias (images/PDF) et une interface de discussion multi-agents pour permettre à plusieurs employés de répondre simultanément sur le même numéro WhatsApp.
